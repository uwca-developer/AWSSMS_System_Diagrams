Key Architectural Features

  • Switcher Architecture Integration: Uses StartChatContact (not StartOutboundChatContact) to enable 211211 shortcode messaging
  • No Phone Number ARN Required: Chat-based contacts don't need a Connect-registered phone number
  • Proper Session Management: Updates both ValidatedCustomers and ConnectChatSessions tables with full participant data
  • Conflict Resolution: Automatically terminates existing sessions before creating new ones
  • Rate Limit Handling: Exponential backoff retry logic for Connect API rate limits (50 TPS)
  • Batch Processing: Step Functions Map state with MaxConcurrency: 2 to prevent overwhelming Connect API
  • Active Session Filtering: Optional pre-campaign filtering to skip contacts already in flows
  • Failed Contact Tracking: Stores failed phone numbers with error details in S3 for retry campaigns
  • All Messages from 211211: Complete integration with Switcher's outbound handler ensures consistent shortcode usage
Note: This architecture replaces the previous StartOutboundChatContact approach, which required a Connect-registered phone number. The new Switcher-based approach creates chat sessions that route through the existing SMS infrastructure, enabling seamless 211211 shortcode messaging for all campaign communications.

Technology Stack

  • Lambda Runtime: Python 3.13
  • AWS Services: Lambda, Step Functions, EventBridge Scheduler, S3, DynamoDB, Connect, ConnectParticipant, End User Messaging (Pinpoint SMS Voice V2)
  • Orchestration: AWS Step Functions with Map state (MaxConcurrency: 2)
  • Storage: S3 (CSVs, failed contacts), DynamoDB (campaigns, sessions, customer data)
  • Messaging: 211211 shortcode via End User Messaging
  • Session Management: 7-day TTL on session records
  • Batch Size: 50 contacts per batch
  • Rate Limiting: Connect StartChatContact: 50 TPS, EUM SendTextMessage: 100 TPS
